# QLU ToolBox 浏览器版整合与 Liquid Glass 设计

## 方案选择

本次评估了三种路线：只替换弹窗皮肤、把桌面端完整搬进扩展、按浏览器场景整合桌面端能力。第一种改动快，但无法形成版本整合；第二种会把 Electron、Python、任务数据库和浏览器下载器塞进扩展，权限、体积与维护成本都不合理。最终采用第三种：保留 Manifest V3 的轻权限架构，复用 GitHub `main` 的品牌资产、成绩导出字段与 5.0 制 GPA 规则，在当前教务会话中完成读取和计算。桌面版继续承担批量任务、登录档案和本地记录，浏览器版专注“用户已经打开教务系统时的一次即时查询”。两者共享品牌和业务定义，但各自遵循平台的最小权限边界。

## 视觉与组件

视觉主题最初定义为“深海数据玻璃”，后续版本已改为森林绿、奶油白与香槟金。弹窗由品牌锁定区、紧凑页面状态提示、当前工具卡和隐私说明组成，不展示尚未定义的功能；成绩浮层由品牌栏、课程/GPA/学期三项指标、操作工具栏和课程列表组成。采用离线可用的本机字体栈，不引入远程字体或图标库。动效全部尊重 `prefers-reduced-motion`，按钮、复选框和对话框均提供可见焦点，浮层支持 Escape 关闭与 Tab 焦点循环。

## 交互与数据流

弹窗打开后查询活动标签页，并继续使用注册表与统一 URL matcher 校验协议、域名、路径和唯一的 `gnmkdm` 参数。若页面不匹配，状态提示仅告知用户手动进入“学生成绩查询”，扩展不会更新或新建标签页。页面匹配后，工具按钮触发 service worker 二次校验并注入成绩模块。成绩模块读取页面当前学年和学期，使用现有登录会话请求同源 Excel，经过大小、ZIP 结构和字段校验后在内存中解析。课程分组完成后，按桌面版规则寻找“总评/总评成绩”，结合学分计算加权 GPA；缺少总评、学分或有效成绩的课程保留展示但不纳入汇总。

## 异常、边界与验证

登录失效、HTML 响应、文件过大、字段变化、空结果、网络错误继续使用不同的可恢复状态，并保持“重新查询”入口。GPA 只是基于已发布总评的即时估算，界面显示参与课程数和学分，避免把不完整数据伪装为精确结论。弹窗只展示已注册并可运行的工具，防止出现空壳功能。验证范围包括 URL 匹配与限制页、5.0 制数字/字母/中文成绩换算、加权 GPA 排除规则、JavaScript 语法、Manifest 权限、PNG 尺寸、扩展 CSP、本地资源引用、确定性 ZIP 打包，以及 400×600 弹窗和宽屏/窄屏成绩浮层的视觉检查。
